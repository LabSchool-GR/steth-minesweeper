# Σ.Τε.Θ. Ναρκαλιευτής

Ανοιχτού Κώδικα Παιχνίδι του Συλλόγου Τεχνολογίας Θράκης.

Το project είναι φτιαγμένο ώστε να μπορεί να χρησιμοποιηθεί σε μάθημα πληροφορικής: ο βασικός κώδικας βρίσκεται σε ένα αρχείο, με ελληνικά σχόλια και καθαρές ενότητες.

## Εκτέλεση

Άνοιξε τον φάκελο `minesweeper` με Godot 4.7 και πάτησε Play.

Από τερματικό μέσα στον φάκελο:

```powershell
..\Godot_v4.7-stable_win64_console.exe --path . --editor
```

## Δημοσίευση

Οι φάκελοι `Demo` και `DemoWindows` περιέχουν έτοιμες εξαγωγές για δοκιμή.
Έχουν `.gdignore`, ώστε το Godot να μην τους διαβάζει ως πόρους του project.

Πριν από νέα δημοσίευση, κάνουμε πάντα νέο export από τα presets:

- `Web` προς `Demo/index.html`
- `Windows Desktop` προς `DemoWindows/steth.exe`

## Γλώσσες

Το παιχνίδι είναι δίγλωσσο:

- Ελληνικά: `locales/el.json`
- Αγγλικά: `locales/en.json`

Κάθε εμφανιζόμενο κείμενο έχει ένα κλειδί, π.χ.:

```json
"game.title": "Σ.Τε.Θ. Ναρκαλιευτής"
```

Ο κώδικας ζητά το κείμενο με τη βοηθητική συνάρτηση `_tr("game.title")`. Έτσι η λογική του παιχνιδιού μένει χωρισμένη από το περιεχόμενο της γλώσσας.

## Πώς παίζεται

- Αριστερό κλικ: άνοιγμα κελιού.
- Δεξί κλικ: σημαία.
- Σε οθόνες αφής, ενεργοποίησε τη λειτουργία `Σημαία` / `Flag`.
- Το πρώτο κλικ είναι πάντα ασφαλές.
- Αν ανοίξουν όλα τα ασφαλή κελιά, ο χρόνος γράφεται στο TOP10.

## Assets

Ο φάκελος `staff` περιέχει:

- `staff/images/logo.png`: λογότυπο STETH για το περιβάλλον της εφαρμογής.
- `staff/images/splash.png`: εισαγωγική οθόνη φόρτωσης για desktop και web export.
- `staff/webfonts/fa-solid-900.woff2`: Font Awesome Solid για εικονίδια σημαίας και βόμβας.
- `staff/webfonts/fa-regular-400.woff2` και `fa-brands-400.woff2`: διαθέσιμα για μελλοντική χρήση.

Οι γραμματοσειρές μπορούν να χρησιμοποιηθούν ως εικονίδια. Για εκπαιδευτική καθαρότητα, τα codepoints δηλώνονται στην αρχή του `scripts/main.gd` ως σταθερές.

Η οθόνη φόρτωσης ορίζεται στο `project.godot`, στις ρυθμίσεις `application/boot_splash`.
Στο web export το Godot τη μετατρέπει αυτόματα στο αρχείο `Demo/index.png`.

## TOP10

Το TOP10 είναι τοπικό και αποθηκεύεται σε:

```text
user://scores.json
```

Κρατάει μέχρι 10 χρόνους ανά επίπεδο δυσκολίας:

- όνομα παίκτη
- δυσκολία
- χρόνος σε δευτερόλεπτα
- ημερομηνία

Στο desktop το `user://` αντιστοιχεί σε φάκελο χρήστη του λειτουργικού. Στο web αποθηκεύεται στον χώρο δεδομένων του browser.

## Ασφάλεια

Για offline `.exe`, το παιχνίδι δεν χρειάζεται δίκτυο και δεν εκτελεί εξωτερικό κώδικα. Τα τοπικά scores είναι κατάλληλα για εργαστήριο, αλλά δεν πρέπει να θεωρούνται απαραβίαστα, γιατί ένας προχωρημένος χρήστης μπορεί να τροποποιήσει το τοπικό αρχείο.

Για web export, προτείνεται single-threaded export. Είναι πιο απλό στο hosting και αποφεύγει την ανάγκη για ειδικά cross-origin headers. Το `.wasm` πρέπει να σερβίρεται με σωστό MIME type `application/wasm`.

Για online leaderboard χρειάζεται ξεχωριστό backend. Δεν βάζουμε ποτέ μυστικά κλειδιά μέσα στο Godot export. Ο server πρέπει να ελέγχει:

- έγκυρη δυσκολία
- λογικό χρόνο
- όριο χαρακτήρων στο όνομα
- προστασία από HTML/script μέσα στα ονόματα
- rate limiting
- HTTPS

## Εκπαιδευτικές ενότητες στον κώδικα

Το `scripts/main.gd` περιλαμβάνει:

- δημιουργία δισδιάστατου πίνακα
- ασφαλές πρώτο κλικ
- τυχαία τοποθέτηση ναρκών
- υπολογισμό γειτονικών ναρκών
- flood fill για άνοιγμα κενών περιοχών
- χειρισμό σημαίας
- συνθήκες νίκης και ήττας
- τοπική αποθήκευση TOP10 με JSON
- δίγλωσσο σύστημα κειμένων με JSON
- χρήση λογότυπου και icon font
